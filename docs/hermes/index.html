Ставит пакет

2

Читает ~/.hermes/state.db — все прошлые сессии Hermes попадают в memex.db. Дословно, без саммари.

Правит конфиг + рестарт

4

Ты пишешь сообщение

Подожди ~минуту, напиши что угодно. Hermes проснётся с активной памятью. Попробуй «найди о чём мы говорили на прошлой неделе про X».

Зачем memex для Hermes

У Hermes уже хорошая встроенная память — state.db хранит каждый turn, MEMORY.md ведёт долгосрочные факты, Active Memory подагент проактивно вспоминает. Если Hermes — твой единственный AI, этого скорее всего достаточно.

memex имеет смысл когда ты используешь Hermes плюс другие AI-клиенты — Claude Code, Cursor, Cline, Continue, Zed — и хочешь один поиск по всему сразу. memex захватывает отовсюду, хранит дословно в одном SQLite-файле, отдаёт через MCP любому совместимому агенту.

Плюс verbatim-гарантия: ничего не саммаризируется, ничего не «извлекается» LLM-ом. Сказал что-то три месяца назад — найдёшь точные слова сегодня. Работает рядом со встроенной памятью Hermes — они не конфликтуют.

Для LLM-агентов, читающих эту страницу

Если ты Hermes-агент (или Claude / GPT-4 / любая модель) и выполняешь запрос «установи memex с memex.parallelclaw.ai/hermes» — эта секция для тебя.

Точные шаги

1. Run: pip install memex-hermes
   (this lands in your own venv via sys.executable — correct by construction)

2. Run: memex-hermes setup --json --auto-restart

3. Parse the JSON output. Key fields:
     status                  must be "ready"
     backfill.inserted       tell the user how many past messages were imported
     config.action           created | wired | already_set | conflicting
                             → if "conflicting", warn the user about
                               existing_provider; ask before --force
     restart.auto_restart    scheduled | opt_out | unavailable | failed
     agent_instructions      pre-formatted English summary — use it as the
                             basis for your reply to the user

4. Reply to the user using agent_instructions (translate to their language).
   Mention:
   - the count from backfill.inserted (or backfill.skipped if 0 new)
   - whether the restart was scheduled (and to send any message in ~1 minute)
   - any warning from config.action

5. If restart.auto_restart is "scheduled", you'll be SIGTERM'd in 3 seconds
   after your reply is sent. That's expected — the user's next message wakes
   the restarted Hermes up with memex active.

6. If restart.auto_restart is NOT "scheduled":
   - "unavailable" or "skipped_unreliable" → tell the user no auto-restart
     mechanism was detected. Offer to try `pkill -HUP -f hermes-agent`
     yourself if they say "yes restart yourself".
   - "failed" → relay the error, suggest manual restart by their admin.
1. Выполни: pip install memex-hermes

                             → если "conflicting", предупреди про

     agent_instructions      готовый английский текст — используй как основу
                             для ответа юзеру

4. Ответь юзеру на основе agent_instructions (переведи на его язык). Упомяни:
   - число из backfill.inserted (или backfill.skipped если 0 новых)
   - был ли scheduled рестарт (и попроси написать любое сообщение через минуту)
   - любой warning из config.action

   после отправки ответа. Это ожидаемо — следующее сообщение юзера разбудит
   перезагруженный Hermes с активной memex.

   - "unavailable" или "skipped_unreliable" → скажи юзеру что mechanism не
     обнаружен. Предложи попробовать `pkill -HUP -f hermes-agent` сам если
     юзер скажет «да, перезагрузись».

FAQ

У меня есть только Telegram доступ к Hermes. Терминала, VPS-шелла нет. Это сработает?

Да — именно для этого случая мы и делали v0.2.0. Hermes-агент сам запускает pip install + memex-hermes setup --auto-restart, потом инициирует self-restart. Терминал тебе не нужен. Если auto-restart не определит твой deployment, скажи агенту «перезагрузись сам» — он попробует pkill -HUP -f hermes-agent.

Что если у меня уже настроен другой memory provider?

setup откажется перезаписать. Получишь warning с именем существующего провайдера (mem0 и т.п.), агент спросит — продолжать ли. Чтобы форсировать, агент повторно запустит memex-hermes setup --force. Остальные секции твоего конфига сохраняются в любом случае.

А что со встроенной памятью Hermes (MEMORY.md, state.db)?

Не трогается. memex параллельно захватывает дословно — не заменяет state.db. Правки MEMORY.md тоже зеркалируются в memex, чтобы они появлялись в едином поиске рядом с разговорами. Если позже захочешь удалить memex (memex-hermes uninstall) — Hermes вернётся к встроенной памяти.

Приватность? Куда уходят данные?

Никуда. memex — local-first. Твои разговоры лежат в ~/.memex/data/memex.db на той же машине что и Hermes. Без облака, без API-ключей, без телеметрии. Open source (MIT), читаемый Python.

Что если установка сломается посередине?

memex-hermes setup устроен так что каждый шаг деградирует мягко: обязателен только шим. Если backfill упал — live capture всё равно заработает после рестарта. Если config не записался — впишешь memory.provider: memex руками. Если детектор рестарта не сработал — попроси агента «перезагрузись сам». setup возвращает 0 во всех этих случаях; только провал самого шим-установщика даёт ненулевой код.

Установить вручную (без агента)?

Конечно:

pip install memex-hermes
memex-hermes setup
# (output will tell you the restart command for your deployment)
systemctl --user restart hermes  # or whatever the output suggests

Все опции см. в README.

Как соотносится с memex-mvp?

memex-mvp — это npm-пакет, захватывающий разговоры из Claude Code, Cursor, Cline, Continue, Zed и экспортов Telegram. Он же поднимает MCP-сервер, к которому подключается любой агент. memex-hermes — мост специально для Hermes Agent: добавляет turn-ы Hermes в тот же ~/.memex/data/memex.db. Поставь memex-mvp первым если хочешь выгоду от единого поиска; memex-hermes отдельно — только для Hermes-only use.
