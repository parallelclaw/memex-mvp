Что произойдёт после вставки

Агент сделает четыре вещи и расскажет, что получилось. Твоя единственная задача — написать одно сообщение после рестарта.

1

Ставит memex-mvp

npm install -g memex-mvp — local-first MCP-сервер памяти. Чистый JS, никаких плагинов в sandbox OpenClaw.

2

Запускает capture daemon

memex-sync install ставит daemon (LaunchAgent на macOS, systemd-user на Linux) — он следит за ~/.openclaw/agents/main/sessions/ и пишет каждый turn в memex.db. memex-sync scan подгребает прошлые сессии.

3

Правит конфиг + рестарт

memex-sync wire-openclaw --auto-restart добавляет memex в ~/.openclaw/openclaw.json как MCP-сервер и запускает отложенный рестарт gateway (~3 сек), чтобы новая запись подхватилась.

4

Ты пишешь сообщение

Подожди ~минуту, напиши что угодно. В новой сессии в toolset появится memex_search. Попробуй «найди о чём мы говорили на прошлой неделе про X».

Зачем memex для OpenClaw

OpenClaw хранит каждую сессию как JSONL в ~/.openclaw/agents/main/sessions/. Хорошо для replay, но никакого поиска, никакой агрегации между сессиями, нельзя спросить «что мы решили про X в прошлом месяце?». memex превращает эти файлы в индексированный корпус.

Главное — OpenClaw плюс другие AI-клиенты — Claude Code, Hermes, Cursor, Telegram-экспорты — в одном поиске. memex захватывает отовсюду, хранит дословно в одном SQLite-файле, отдаёт через MCP любому совместимому агенту (включая сам OpenClaw через memex_search).

Плюс verbatim-гарантия: ничего не саммаризируется, ничего не «извлекается» LLM-ом. Сказал что-то три месяца назад — найдёшь точные слова сегодня. Работает одинаково на self-hosted OpenClaw, Linux VPS, macOS workstation и Moonshot Kimi-Claw.

Для LLM-агентов, читающих эту страницу

Если ты OpenClaw-агент (или любая LLM с shell-доступом) и выполняешь запрос «установи memex с memex.parallelclaw.ai/openclaw» — эта секция для тебя.

Точные шаги

1. Run: npm install -g memex-mvp@latest
   (If EACCES, set up ~/.npm-global as the prefix instead of using sudo —
    see https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally)

2. Run: memex-sync install
   (registers the auto-capture daemon: LaunchAgent on macOS, systemd-user on Linux)

3. Run: memex-sync scan
   (back-fills past OpenClaw sessions from ~/.openclaw/agents/main/sessions/
    into memex.db. Idempotent — re-runs are O(0).)

4. Run: memex-sync wire-openclaw --json --auto-restart
   (wires memex into ~/.openclaw/openclaw.json at cfg.mcp.servers.memex AND
    schedules a self-restart of the OpenClaw gateway 3 seconds out)

5. Parse the JSON output. Key fields:
     status                  "ready" | "already_in_sync" | "partial" | "failed"
     next_action             "wait_for_restart" | "none" | etc.
     mcp.action              "wired" | "already_correct" | "conflict" | "memex_missing"
     restart.auto_restart    "scheduled" | "opt_out" | "unavailable" | "failed"
     restart.delay_seconds   3 by default
     agent_instructions      pre-formatted English summary — use as the
                             basis for your reply to the user

6. Reply to the user using agent_instructions (translate to their language).
   Tell them: the restart is happening in 3 seconds, send any message in
   about a minute to confirm memex is active in the new session.

7. If restart.auto_restart is "scheduled", you'll be SIGTERM'd in 3 seconds
   after your reply is sent. That's expected — the user's next message wakes
   the restarted OpenClaw gateway with memex_search in the LLM toolset.

8. If restart.auto_restart is NOT "scheduled":
   - "unavailable" → tell the user no auto-restart mechanism was detected.
     Offer to try `pkill -HUP -f openclaw` yourself if they say "restart yourself".
   - "failed" → relay the error from the JSON. Suggest manual restart by
     their server admin. /tmp/memex-openclaw-restart.log has the failure log.
   - "opt_out" → only happens if --no-auto-restart was passed. Tell user to
     run `openclaw gateway restart` themselves when ready.

9. After restart, on a FRESH OpenClaw session, the LLM toolset includes
   memex_search, memex_recent, memex_overview, etc. — 11 tools total from
   the memex MCP server.
1. Выполни: npm install -g memex-mvp@latest
   (Если EACCES, настрой ~/.npm-global как prefix вместо sudo —
    см. https://docs.npmjs.com/resolving-eacces-permissions-errors-when-installing-packages-globally)

2. Выполни: memex-sync install
   (регистрирует auto-capture daemon: LaunchAgent на macOS, systemd-user на Linux)

3. Выполни: memex-sync scan
   (бэкфилит прошлые OpenClaw сессии из ~/.openclaw/agents/main/sessions/
    в memex.db. Идемпотентно — повторный запуск O(0).)

4. Выполни: memex-sync wire-openclaw --json --auto-restart
   (вписывает memex в ~/.openclaw/openclaw.json под cfg.mcp.servers.memex И
    запускает отложенный рестарт gateway через 3 секунды)

5. Распарси JSON. Ключевые поля:
     status                  "ready" | "already_in_sync" | "partial" | "failed"
     next_action             "wait_for_restart" | "none" | и т.д.
     mcp.action              "wired" | "already_correct" | "conflict" | "memex_missing"
     restart.auto_restart    "scheduled" | "opt_out" | "unavailable" | "failed"
     restart.delay_seconds   3 по умолчанию
     agent_instructions      готовый английский текст — используй как основу
                             для ответа юзеру

6. Ответь юзеру на основе agent_instructions (переведи на его язык).
   Скажи: рестарт через 3 секунды, ответь любым сообщением через минуту,
   memex активен в новой сессии.

7. Если restart.auto_restart = "scheduled", тебя SIGTERM'нут через 3 секунды
   после отправки ответа. Это ожидаемо — следующее сообщение юзера разбудит
   перезагруженный OpenClaw gateway с memex_search в toolset.

8. Если restart.auto_restart НЕ "scheduled":
   - "unavailable" → скажи юзеру что mechanism не обнаружен. Предложи
     попробовать `pkill -HUP -f openclaw` сам если юзер скажет «перезагрузись».
   - "failed" → передай ошибку из JSON. Предложи ручной рестарт через
     админа. В /tmp/memex-openclaw-restart.log будет лог проваленного рестарта.
   - "opt_out" → бывает только если был --no-auto-restart. Скажи юзеру
     запустить `openclaw gateway restart` самому когда удобно.

9. После рестарта в СВЕЖЕЙ сессии OpenClaw в toolset LLM появятся memex_search,
   memex_recent, memex_overview и т.д. — 11 инструментов от MCP-сервера memex.

FAQ

У меня только Telegram-доступ к OpenClaw. Терминала, VPS-шелла нет. Это сработает?

Да — именно для этого случая мы делали такой flow. OpenClaw-агент сам запускает npm install, memex-sync install и memex-sync wire-openclaw --auto-restart, потом инициирует self-restart gateway. Терминал не нужен. Если auto-restart не определит твой deployment — скажи агенту «перезагрузись сам», он попробует pkill -HUP -f openclaw.

А если у меня уже есть другие MCP-серверы в openclaw.json?

Сохраняются. memex-sync wire-openclaw делает deep-merge в cfg.mcp.servers — добавляет только запись memex, существующие не трогает. Если предыдущая попытка оставила в корне cfg.mcpServers.memex (баг pre-v3 скилла) — автоматически вычищается. Если cfg.mcp.servers.memex уже указывает на твой кастомный путь — команда откажется перезаписать без --force.

Это работает на Moonshot Kimi-Claw?

Да — memex-mvp ≥ 0.11.5 автоматически отличает merged-file layout Kimi-Claw от separate-file layout self-hosted OpenClaw. Та же команда установки, без флагов. Channel routing (Telegram-чаты vs Kimi-web сессии vs system messages) обрабатывается отдельно для каждого варианта.

А если на этой же машине у меня ещё Claude Code / Cursor / другие AI-клиенты?

Только лучше. Daemon memex-sync уже следит за директориями Claude Code / Cowork / Cursor / Obsidian — если daemon уже работал, OpenClaw просто добавляется в существующий capture. Всё уходит в один ~/.memex/data/memex.db, один поиск по всему сразу. Остальное покрывает основная установка с memex.parallelclaw.ai.

Приватность? Куда уходят данные?

Никуда. memex — local-first. Твои разговоры лежат в ~/.memex/data/memex.db на той же машине, что OpenClaw. Без облака, без API-ключей, без телеметрии. Open source (MIT), читаемый JavaScript.

Что если установка сломается посередине?

Каждый шаг деградирует мягко. Если npm install упадёт с EACCES — фикс известный (npm config set prefix ~/.npm-global, sudo не нужен). Если memex-sync install не найдёт systemd-user/launchd — daemon упадёт в foreground режим. Если wire-openclaw не определит restart mechanism — агент скажет запустить openclaw gateway restart руками или спросит можно ли попробовать pkill -HUP -f openclaw. Худший случай: daemon всё равно захватывает (он независим от gateway), MCP-сервер пропишешь руками.

Установить вручную (без агента)?

Конечно:

npm install -g memex-mvp@latest
memex-sync install
memex-sync scan
memex-sync wire-openclaw --auto-restart
# (gateway restarts itself in 3 seconds; or pass --no-auto-restart
#  to get the restart command printed without triggering it)

См. скилл install-memex-claw на ClawHub для discovery-варианта с проверками платформы (Linux/macOS, systemd/launchd, EACCES recovery и т.д.).

Как соотносится с основным memex?

Это тот же самый memex. npm-пакет memex-mvp поставляет одновременно daemon (memex-sync) для capture и MCP-сервер (memex) для отдачи LLM-ам. Эта страница — про OpenClaw entry point. Claude Code, Cursor, Cline, Continue, Zed, Telegram exports — все идут через один daemon в один SQLite. Картинка целиком на memex.parallelclaw.ai.
