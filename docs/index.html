How is memex different from native Claude / ChatGPT memory?

У нативной AI-памяти пять структурных дыр. Memex закрывает все пять:

  • Жёстко изолированные per-project silos (Claude Memory закрывает границы намертво) → memex project-aware, не siloed: опциональный project-фильтр в поиске, default — широкий. Граница на каждый запрос — твой выбор.
  • Между продуктами Anthropic нет мостов (Web Claude не видит Code; Code не видит Cowork) → все продукты Anthropic + субагенты Cowork в одном поисковом индексе.
  • Cross-AI памяти не существует by design (ChatGPT, Cursor, Cline, Zed, Telegram-боты — каждый держит свой silo) → любой MCP-клиент читает одну и ту же память через 11 стандартных тулзов.
  • Привязка к одной машине (история Claude Code на конкретном ноуте) → memex — обычный SQLite-файл, синк через iCloud / Syncthing / rsync на твой вкус.
  • Аккаунт умирает — память умирает (Anthropic блокирует без предупреждения; country policy режет RU/CN/IR/KP) → SQLite-файл на твоём ноуте не зависит от провайдера. Твоё by architecture, не по policy.

Источники: anthropics/claude-code #36561, #39195, #25739; HN #47853021 (тред про апрельские баны 2026).

What do I need to install memex? Any limitations?

Жёсткие требования (без них memex не запустится):

  • Node.js 20.x – 24.x (рекомендуем 22 LTS). Поставь через nodejs.org, brew install node или nvm.
  • macOS 12+ или Linux с inotify. Windows — только через WSL.
  • Xcode Command Line Tools на macOS (xcode-select --install) — нужны для нативной сборки better-sqlite3, если для твоей Node-версии нет prebuilt binaries.
  • MCP-совместимый AI-клиент — Claude Code, Cursor, Cline, Continue, Zed. Без него memex строит индекс, но обращаться к нему будет некому.

Известные ограничения:

  • ❌ Web-only AI (ChatGPT в браузере, Claude.ai web) — эти сессии живут на серверах вендора, на твоём диске их нет.
  • ❌ Мобильные AI-приложения (ChatGPT iOS, Claude Android) — phone-data не пишется на твой компьютер.
  • ❌ Сессии на VPS / в облаке — memex читает только локальную файловую систему.
  • ❌ Auto-capture daemon на Linux — npx memex-sync install работает только на macOS (LaunchAgent). На Linux запускай npx memex-sync в foreground или сделай свой systemd unit.
  • ❌ Telegram mobile export — мобильный TG не умеет экспортировать чаты. JSON для memex'a делает только Telegram Desktop.

✅ Положительное «ограничение»: Internet не нужен после установки. Никаких phone-home, никаких API-ключей, никаких облачных зависимостей. Это feature, не bug.

Across multiple devices — does memex work?

Memex живёт на одной машине — daemon ловит локальные файлы, SQLite строится локально. Три паттерна для multi-device сегодня:

  • Синк SQLite-файла. ~/.memex/data/memex.db — обычный файл, реплицируй через iCloud / Syncthing / rsync. Daemon пишет на основной машине, остальные читают через свой локальный memex MCP. Один writer, много readers.
  • Memex на каждом устройстве независимо. Установи отдельно на каждый ноут. Нет синка, нет конфликтов — но память не унифицирована.
  • Mobile через Telegram-бот (в roadmap'е, код написан). Пересылаешь сообщения в @memex_bot — бот пишет JSON в ~/.memex/inbox/ основной машины.

iCloud setup на macOS:

export MEMEX_DIR="$HOME/Library/Mobile Documents/com~apple~CloudDocs/memex"

⚠ Только один writer. Auto-capture daemon (memex-sync) запускай ровно на одной машине, остальные только читают. Параллельные writer'ы через filesystem sync могут корраптнуть WAL.

What about privacy and encryption?

✅ Что memex делает: Живёт только на твоей машине, никуда не звонит, без API-ключей, без network access. OS-level права — читает только твой user.

❌ Что не делает: Не шифрует БД, не редактирует секреты которые ты вставлял в чаты, нет пароля на сам memex.

📦 Не добавляет риск — концентрирует. Твоя AI-история уже на диске в plain text (Claude Code JSONL, Cursor state.vscdb, Cowork files, Obsidian markdown, Telegram local DB). Memex консолидирует в один SQLite. Те же данные, в одном месте вместо пяти.

🛡️ Топ-рекомендация: FileVault. На macOS: System Settings → Privacy & Security → FileVault → Turn On. Full-disk AES-256 на уровне OS — закрывает ~80% реалистичных угроз (украденный ноут, кража backup, malware без root). На Linux то же делает LUKS.

What about phone / web AI?

Сегодня: memex ловит только то что на твоём ноуте. Phone-based AI (ChatGPT mobile, Claude iOS) и web-only сессии (Claude.ai web, ChatGPT в браузере) не пишут файлы тебе на диск — memex'у нечего читать.

Workaround'ы: Для Telegram-ботов — ручной export → дроп в inbox. Для ChatGPT — export данных через Settings, парсер в roadmap'е.

Roadmap: Telegram-бот (код написан в bot/) для mobile capture — пересылаешь мысли и сообщения с телефона, индексируется автоматически на ноуте.

Moving to a new laptop — does memex come with me?

Да. ~/.memex/data/memex.db — обычный SQLite-файл. Три команды на старом ноуте, три на новом — вся история переезжает вместе с FTS5-индексом. Точные команды — в README → Миграция между устройствами.

What if my Claude / ChatGPT account gets banned?

Memex не зависит от твоего provider-аккаунта — читает файлы которые уже на твоём диске. Даже после бана твои Claude Code сессии в ~/.claude/projects/ остаются на месте. Memex индексирует их и отдаёт через MCP любому локально доступному AI (Llama, Qwen, или Claude через другой аккаунт). Твоя работа переживает канал доступа.

DB opens read-only — safe to run while memex-sync daemon is writing. When called without arguments (memex), the binary still runs as MCP stdio server — that's the mode Claude Code, Cursor, Cline launch via their config files. CLI and MCP modes coexist in one package.
